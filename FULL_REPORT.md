# Полный отчет по заданиям 2 и 3

## Тема работы

Электронный дневник медицинского самоконтроля (ЭДМС).

---

# Задание 2. Проектирование базы данных

## Цель работы

Спроектировать реляционную базу данных для системы электронного дневника медицинского самоконтроля.

Система должна обеспечивать:

- хранение информации о пациентах и врачах;
- фиксацию действий и симптомов;
- хранение результатов самоконтроля;
- формирование отчетов.

---

# Описание предметной области

Система ЭДМС используется пациентами и врачами для ведения медицинского самоконтроля.

Пациент фиксирует:

- действия;
- симптомы;
- пульс;
- давление;
- температуру;
- субъективную оценку самочувствия.

Врач может анализировать динамику состояния пациента и формировать отчеты.

---

# Связи между таблицами

- Один врач может иметь несколько пациентов.
- Один пациент может иметь много записей самоконтроля.
- Одно действие может использоваться в разных записях.
- Один симптом может встречаться в нескольких наблюдениях.
- Для одного пациента может быть сформировано несколько отчетов.

---

# Реализованные SQL-запросы

## Отчет 1

Динамика объективного показателя за выбранный период.

Пример:

- изменение пульса пациента за период времени.

## Отчет 2

Три даты наиболее плохого самочувствия.

Используется минимальное значение `wellbeing_score`.

## Отчет 3

TOP-3 действий по средним значениям пульса и давления.

Используется группировка записей по действиям.

- работа за компьютером.

| список действий | actions |
| форма добавления | actions |

## Основные функции

- просмотр списка;
- добавление действия;
- редактирование;
- фильтрация;
- поиск.
